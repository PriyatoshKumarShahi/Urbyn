Enhance Urbyn with a full accountability and SLA system so every civic issue becomes a measurable, time-bound responsibility instead of a passive complaint. For each issue, automatically assign a responsible department from category rules, generate an SLA deadline based on issue type and severity, and persist timestamps for reported, assigned, in-progress, and resolved states. Extend the schema with department, priorityScore, slaDeadline, isOverdue, overdueByHours, resolvedAt, resolutionTimeHours, and a statusHistory array that records actor, previous status, new status, note, and timestamp. Build backend utilities that compute deadlines on create, recompute overdue state on reads, calculate resolution time on resolve, and expose admin analytics for department-wise average resolution time, within-SLA percentage, overdue count, and a simple accountability score. On the frontend, add a timeline component to every issue card, show countdown labels like “12h left” or “Overdue by 2d”, use strong visual status chips, and highlight ignored issues in feed, dashboard, and hotspot views. In the admin dashboard, include sortable columns for department, SLA deadline, overdue state, resolution time, and reporter, and open a side panel with full issue details, images, history, and direct link to the issue card. The UI should stay responsive, light themed, and neubrutalist, with charts showing SLA compliance and department performance for hackathon demos.
